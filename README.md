  💻 Operating System Lab Programs
 
This repository contains basic Operating System laboratory programs written in the C programming language. The programs demonstrate common CPU scheduling techniques and important OS concepts used in academic coursework.
 
   📌 Repository Overview
 
The goal of this repository is to store and organize Operating System lab experiments implemented using C. These programs help students understand how CPU scheduling algorithms and core operating system mechanisms work internally.
 
   🖥️ Programming Language
 
All programs are written in the C programming language.
 
   ⚙️ Requirements
 
To compile and run these programs you need:
 
• GCC Compiler  
• Terminal (Linux / Windows / macOS)  
• Basic understanding of C programming
 
   🚀 Compilation and Execution
 
1. Compile the program
 
gcc filename.c
 
2. Run the executable
 
./a.out
 
Example:
 
gcc priority.c  
./a.out
 
   📂 Programs in this Repository
 
• Priority Scheduling Algorithm  
• First Come First Serve (FCFS) Scheduling  
• Shortest Job First (SJF) Scheduling  
• Round Robin Scheduling  
• Additional OS algorithms (to be added later)
 
   🧪 Example Implementation
 
The Priority Scheduling program calculates the following:
 
• Waiting Time  
• Turnaround Time  
• Order of Process Execution

This repository is created for Operating System laboratory exercises and learning purposes.
